# Actor / gimmick editor-coverage audit

Generated by `MegatonHammer --actoraudit both`. It walks **every scene and room** of both games
(OoT 101 scenes, MM 102 scenes), collects the distinct set of actor types actually placed, and
cross-references each against:

1. **The actor name database** — does the editor show a real name, or "Obsolete 0xNNNN"?
2. **The typed param schema** — does it get Hammer-style SmartEdit fields for its switch/chest/flag
   logic (vs. a raw 16-bit params hex value, which is always editable regardless)?

Full per-actor tables are in [actor-coverage.txt](actor-coverage.txt).

## Param schemas (SmartEdit) — by design a curated subset

A typed param schema is the Hammer "SmartEdit" layer: it decodes an actor's 16-bit params into named
fields (chest contents, switch flag, drop type, transition index, flag I/O roles). It exists for the
**gimmick/logic** actors that benefit from it — chests, doors, switches, pots/crates/collectibles,
water-level controllers, the Dark Link room, the Stone Tower inverter, trigger regions, etc. The
remaining ~300 actors are scenery, enemies, and ambient props whose params are simple or
position-only; they keep the always-available raw params editor. Schemas can be added incrementally;
the audit report's "SmartEdit candidates" section ranks the schema-less actors by placement count to
guide that.

## How to re-run

```
MegatonHammer --actoraudit both      # or: oot | mm
```

Writes `actor-coverage.txt` next to the executable and prints a one-line summary per game.
